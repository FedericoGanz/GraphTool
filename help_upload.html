General

  • Data: country-year panel.
  • File formats: .txt, .xls, .xlsx, .csv (separated by commas or semicolons), .dta

Variables that must be included

  1. 'Var_name': variable name.
  2. 'Units': units in which the variable is expressed.
  3. 'Ctry_iso': country ISO 3 code (see 'Countries/aggregates WDI' tab in 'Data') .
  4. 'Year': year (non-numeric values will be treated as missing).
  5. 'Value': value (non-numeric values will be treated as missing).

Additional variables that can be included

  1. 'Database': source of the data. If not included, then the file name is used. This is the variable used in plots if "Include source" option is selected.
  2. 'Var_code': variable code. It should be unique for each variable. If not included, then 'Var_name'_'Database' is used.
  3. 'Description' (Metadata): variable description.
  4. 'Source_data' (Metadata): database where the information was retrieved.
  5. 'Source_org' (Metadata): organization that collects the data.

Data loading process

When loading, data will be filtered using the country selection (countries/aggregates included in the vector defined by the seven country input variables - Analyzed, Structural, Aspirational, Groups of countries, Aggregates, All countries and All aggregates) and the Year range selected at the moment of updating.

If, for example, country selection is empty at the moment of updating, then no data will be loaded.

Countries and aggregates which do not have a ISO 3 code in the WDI database will not be included.

After the data is loaded, changes in Year range will not affect the external data already loaded (it will only affect WDI data).

However, both changes in Periods (name or limits) and Ctry_group will affect the external data that has been loaded.

For example: imagine we have a dataset with data for Argentina, and that Argentina was not initially included in the country selection. Moreover, imagine we include, after loading, Argentina in the Structural group. The result is that Argentina will not appear in the external data of the dataset.

However, imagine now that Argentina was initially included in the country selection as the Analyzed unit, and that, after loading, we change it to the Aspirational group. In this case Argentina will be included in the dataset and the variable Ctry_group will indicate the new classification (Aspirational).

The uploaded dataset will have as many rows as 'variables * countries * years' selected.

If, for example, the dataset we want to upload contains only one country (Argentina), one variable (GDP in USD) and one year of data (2020), and we set the All countries and All aggregates options as True (304 countries/aggregates) and the Year range to 2000-2019 (20 years), then the final dataset will include 6,080 rows (1 * 304 * 20). However, only one row will have no empty data for the column Value.

Note: a variable is a unique Var_name/Var_code/Units/Database.

For example, if the dataset we are trying to upload includes a row with the following values: Var_name: 'GDP'; Var_code: 'gdp_usd'; Units: 'USD'; Database: 'National sources', and another row with the same values for all variables but for Database (for example, 'IMF' instead of 'National sources'), then the dataset has at least two variables.
